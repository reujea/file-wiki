# Lesson Learned: Phase 101 내부 코드명 UI 노출 제거 + Pipeline 이관 검토

## 상황

2026-05-26 Phase 101. 사용자 질문 "C1은 무슨뜻이야?" — 본인 GUI 사용 중 Settings 탭의 "🧠 자동 추천 (C1)" / "⚙️ C1 자동 추천 임계값" 라벨에서 의미 불명 표기 발견.

C1은 외부 프로젝트 ruvnet/ruflo의 카테고리 코드 (A=효율 / B=품질 / C=학습보안). 본 프로젝트 lesson 30 흡수 시 Ruflo 명명을 그대로 차용. **내부 추적용 코드명이 UI 라벨에 노출됨**.

직전 메모리 `feedback_no_phase_in_ui.md` (Phase 76 사용자 직접 피드백 기록): "UI 텍스트에 Phase 번호 노출 금지". 같은 메타 패턴의 변형 사례.

직후 사용자 요청: "Settings 운영/이벤트 훅의 설정들을 Pipeline 메뉴 탭별 항목으로 이관 가능 검토" — 본 phase 본 작업 + 검토 진행.

## 문제

### 문제 1 — 내부 코드명 (C1) (C2) UI 노출 4건

`src/ui/index.html`:
- L268: `<h4>🧠 자동 추천 (C1)</h4>`
- L290: `<option>자동 추천 (C1)</option>`
- L306: `<h4>⚙️ C1 자동 추천 임계값</h4>`
- L321: `<h4>🔒 PII 검출 패턴 (C2)</h4>`

`src/ui/dashboard.js`:
- L1683: `<p>자동 롤백이 아닌 사용자 검토 권고 시스템 (Phase 92 H1).</p>` — Phase 번호도 노출
- L2298: `description: '자동 추천 · C1 임계값 · PII · MCP 카탈로그'` — 좌측 네비 설명
- L2170 / L4201 / L4204: 에러 메시지 "C1 임계값 로드 실패" / "C1 ${key} 저장" / "C1 저장 실패"

총 **8건 UI 노출**. 사용자는 "C1" 의미 불명 → 본 phase 질문 트리거.

### 문제 2 — Pipeline 탭 이관 검토 결과 (구현 0건)

사용자 요청: Settings 운영(5 카드) + 이벤트 훅(1 카드) → Pipeline 탭 노드 인스펙터로 이관.

검토 결과:
| 카드 | 이관 가능성 | 사유 |
|------|---------|------|
| 자동 추천 / 임계값 | 🔴 불가 | 시스템 전역 누적 분석 도구, 노드 매핑 불가 |
| MCP 카탈로그 | 🔴 불가 | Pipeline 외부 인터페이스 |
| PII 마스킹 | 🟢 가능 | 검색 deliver 노드 매핑 |
| PII 패턴 | 🟡 부분 | 가공+검색 2 노드 분산 = 메타 룰 1 위반 위험 |
| 이벤트 훅 | 🟡 부분 | 5 이벤트 노드 분산 = HookDefinition 통합 관리 손실 |

**3 옵션 제시** (A 이관 안 함 / B PII 마스킹 1건만 / C PII+이벤트 훅) → 사용자 결정 영역 (메타 룰 22).

본 phase는 검토 결과 보고만, 구현은 별도 phase (사용자 결정 후).
