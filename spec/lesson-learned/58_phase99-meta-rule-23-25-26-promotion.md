# Lesson Learned: Phase 99 메타 룰 23/25/26 정식 승격 + 메타 룰 19 자기 위반 해소

## 상황

2026-05-26 Phase 99 진행. Phase 98 종결 시 후보 7건 누적(21/22/23/24/25/26/27) → 메타 룰 23 후보(승격 기준 정형화) 우선순위 최상으로 식별. 사용자 명시 "Q1 진행" → 메타 룰 23 정형화 진행.

진행 중 메타 룰 19 자기 위반 발견 — 후보 22/24/25/26/27이 모두 lesson 본문에만 분산 등록, META.md 단일 진실원 등재 누락.

## 문제

### 문제 1 — 메타 룰 후보 7건 누적, 승격 기준 부재

Phase 91~98 동안 메타 룰 후보 21~27 누적. 각 후보가 "1건 추가 시 META 등록 검토" 표기되어 있으나:
- 객관 기준 부재 — 누적 N건, 어떤 N? 1건이라도 명확하면? 3건? 5건?
- 후보 인플레이션 위험 — 7건 후보 평가 비용 증가
- 정식 승격 시점 일관성 부재 — 메타 룰 19(5건) / 20(4건) / 18(3건) / 13(1건) 모두 다른 기준

### 문제 2 — 메타 룰 19 자기 위반 발견

META.md 단일 진실원 위임 정식 승격(Phase 94) 이후 5건 누적 자기 적용. 그러나:
- 후보 22 본문 → lesson 51 / 53 / 55 / 56 / 57 분산
- 후보 24 본문 → lesson 54 / 56 / 57 분산
- 후보 25 본문 → lesson 55 / 56 분산
- 후보 26 본문 → lesson 56 분산
- 후보 27 본문 → lesson 57 분산

**META.md는 인덱스 + 정식 룰만 보유** 정책에 따라 후보는 lesson 본문에만 있었음. 이는 결과적으로 메타 룰 19 자기 위반 — 단일 진실원이 lesson 본문에 분산.

### 문제 3 — 메타 룰 25 후보 등재 시 체크리스트 미명시

Phase 96 lesson 55에서 메타 룰 25 후보 등록 시 "자기 적용 의무" 핵심만 등록. 신규 작업 시 사전 체크리스트 미작성 → 메타 룰 23 §승격 3요소 중 (2) 미충족.

## 원인

### 직접 원인
1. (문제 1) 메타 룰 13(1건만으로 정식) ↔ 메타 룰 19(5건) 등 정식 승격 시점이 케이스별 즉흥 결정. 메타 룰 23 후보가 lesson 53(Phase 94)에서 등록되었으나 본인 자기 적용 자체가 지연
2. (문제 2) META.md 본문 정책이 "정식 룰만"이었음. 후보는 lesson 본문이 적합하다고 판단했으나 메타 룰 19 자기 위반 인지 못 함
3. (문제 3) 메타 룰 25 등록 시 "정식 승격 시 즉시 자기 적용"만 명시하고 체크리스트 구체화 미실시

### 구조적 원인
- 메타 룰 인플레이션 = 메타 작업 비용 증가. 메타 룰 23 부재로 후보가 7건까지 누적되며 평가 비용 ↑
- 메타 룰 자체에 메타 룰 19/25 자기 적용 패턴이 적용되어야 한다는 메타-메타 패턴 (Phase 99 자기 적용으로 해소)
- META.md "본문이 아닌 인덱스" 정책과 메타 룰 19 단일 진실원이 충돌 — 정식 룰만 본문은 단일 진실원 적용 영역이 좁아짐
