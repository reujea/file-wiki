# Lesson Learned: Phase 89 — 내부 진행 가능 트리거 소진과 외부 신호 의존 단계 진입

Phase 89가 단일 phase에 9개 영역(N-3 / N-4 / A1 측정 / C2 측정 / #6 HyDE / C-1/C-2/C-3 / D 메타 룰 / B-2 Sparse 인프라 / B-1 5변형 재측정)을 묶어 진행. 회귀 0 + clippy 0 + lib 343 유지 통과. 그러나 phase 종료 시점에 **남은 모든 트리거 대기 항목이 외부 사용자 신호 또는 도메인 확장 의존**임이 확인됨.

## 문제

1. **자동 진행 가능 항목이 단일 phase에 모두 소진** — 다음 phase 진로가 사용자 결정 또는 외부 신호에 의존
2. **단일 phase 9영역 묶음의 검증 비용** — 영역 간 의존이 없는데도 한 phase에 묶이면 회귀 추적 시 영역 분리 어려움
3. **자동 측정 가능 항목 vs 자동 측정 불가 항목의 경계 명확화 부재** — A2-def / B1-def는 "검색 만족도"라 자동 측정 불가지만, 이 판단을 phase 진입 전 명시한 적이 없어 phase 중간에 발견

## 원인

직접 원인:
- 권장 우선순위 1단계(N-3/N-4) → 2단계(측정) → 3단계(외부 트리거) → C/D/B 영역까지 한 세션에서 모두 진행하라는 사용자 지시 ("B,C,D 진행해"). 9영역 묶음이 사용자 지시의 자연스러운 결과
- 외부 신호 의존 항목과 내부 측정 가능 항목을 사전 분류하지 않고 trigger 표만 보고 진입

구조적 원인:
- lesson 14 패턴(인프라 토대 누적) + lesson 30 패턴(측정 후 활성화)의 누적 효과. 인프라가 모두 완료된 상태에서 "측정만 남았다"는 표기가 실제로는 "측정도 자동화 불가능한 검색 만족도"를 포함
- META.md 메타 룰에 "자동 측정 가능 여부" 사전 분류 규칙 부재

## 개선

### 새 메타 룰 (메타 룰 16): 자동 측정 가능성 사전 분류 — ✅ Phase 90에서 META.md 정식 승격

각 트리거 대기 항목에 다음 라벨 추가 필수:

- 🟢 **자동 측정 가능**: 코퍼스 + 벤치 함수만 있으면 진행 (예: #2 / #4 / A1 hit / C2 fp)
- 🟡 **사용자 코퍼스 의존**: 특정 도메인 코퍼스가 있어야 신호 발생 (예: #4 다른 도메인 / #8 표 비중 / 다른 PII 도메인)
- 🔴 **사용자 만족도 의존**: 자동 측정 불가, 실 사용자 피드백 필요 (예: #6 HyDE / A2-def / B1-def / #7 MRR 회귀 체감)

phase 진입 전 분류 확인. 🔴은 phase 시작 전 "외부 신호 대기" 단계로 분리.
